# Toolforge builds API

API to start builds of images to use in toolforge.

# Developing
## If you modify the swagger definition
The core of the app is generated by go-swagger, from the swagger yaml file, so every time you modify the swagger yaml definiton, you have to regenerate the code, to do so just:
```
> make gen-api
```

Then you can manually modify the code under `cmd/builds-api` to match the new changes if necessary.


# Building the docker image

You can just:
```
> make image
```
That will create a `toolforge-builds-api:dev` image locally, that you can then re-tag and push if you want.


# Updating the code after changing the swagger yaml file

## On the api side
```
> make gen-api
```
Then you might have to tweak `cmd/secrets-api/main.go` to add any new handlers, remove old ones or change any if needed.

## On the client side
This one is a bit more tricky, as more changes were needed there.
NOTE: It's easier if you don't have any local changes to any other go files, commit/stash them if you do.

First get the files that were manually modified:
```
> git grep MANUALLY
```

Then generate the new code (note that it will not delete leftover files if any):
```
> make gen-api
```

# Testing locally

## Deploy locally

Prerequisites:
* Minikube
* Toolforge [Cert-manager](https://gitlab.wikimedia.org/repos/cloud/toolforge/cert-manager)
* Toolforge [api-gateway](https://gitlab.wikimedia.org/repos/cloud/toolforge/api-gateway)
* Toolforge [buildservice](https://github.com/toolforge/buildservice)

First, you have to enable the minikube docker environment:
```
> eval $(minikube docker-env)
```

Then build the image:
```
> make image
```

And deploy:
```
> ./deploy.sh
```

If you only want to deploy the golang code, you can build the image in minikube docker env:
```
> eval $(minikube docker-env)
> make image
```

And then force a rollout on the deployment:
```
> kubectl rollout restart -n builds-api deployment builds-api
```

## Access locally
With this, you can start a tunnel to expose the api gateway with:
```
> minikube service api-gateway -n api-gateway
```

And do a curl request:
```
> curl --cert ~/.minikube/profiles/minikube/client.crt --key ~/.minikube/profiles/minikube/client.key --insecure 'https://192.168.49.2:30003/builds/v1/healthz'
```

Note that you will have to run a build to see it's logs :)
